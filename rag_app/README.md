# RAG · Document Q&A

A production-structured **Retrieval-Augmented Generation** app built with Streamlit, OpenAI, and Qdrant.

---

## Architecture

```
rag_app/
├── app.py                        ← Presentation layer (Streamlit UI)
├── requirements.txt
├── .env.example                  ← Copy to .env and fill in keys
│
├── config/
│   └── settings.py               ← Centralised configuration (dataclasses + dotenv)
│
├── business/                     ← Business logic layer
│   ├── document_processor.py     ← PDF text extraction & chunking
│   ├── embeddings.py             ← OpenAI Embeddings API wrapper
│   └── rag_service.py            ← RAG orchestration (retrieve + generate)
│
└── persistence/                  ← Data layer
    └── vector_store.py           ← Qdrant wrapper (upsert, search, reset)
```

---

## Quick Start

### 1. Install dependencies
```bash
pip install -r requirements.txt
```

### 2. Configure environment
```bash
cp .env.example .env
# Edit .env and add your OPENAI_API_KEY
```

### 3. Run
```bash
streamlit run app.py
```

## Environment Variables

| Variable | Required | Description |
|---|---|---|
| `OPENAI_API_KEY` | ✅ Yes | Your OpenAI secret key (`sk-…`) |
| `QDRANT_URL` | ⬜ Optional | Qdrant Cloud cluster URL. Blank = in-memory mode |
| `QDRANT_API_KEY` | ⬜ Optional | Qdrant API key (only needed with `QDRANT_URL`) |

---

## How It Works

1. **Upload PDFs** via the sidebar → text is extracted with `pdfplumber`
2. **Chunking** splits text on Markdown headers / paragraph breaks with configurable `max_words` and `overlap_words`
3. **Embeddings** are generated in batches using `text-embedding-3-small`
4. **Vectors** are stored in Qdrant (in-memory or cloud)
5. **Query** → question is embedded → top-5 chunks retrieved by cosine similarity
6. **Answer** generated by `gpt-4o-mini` using retrieved context as grounding
